# Roco Box Detector 使用说明

## 快速开始

1. 解压后双击 `RocoBoxDetector.exe`
2. 屏幕变暗，用鼠标拖拽框选战斗时惊喜盒子出现的区域，可以先进入一场战斗再确认框选位置，按 `Enter` 确认
3. 检测自动开始，屏幕右侧出现识别面板

## 识别面板按钮

面板顶部的功能按钮，从左到右：

| 按钮 | 名称 | 功能 |
|------|------|------|
| `◌` / `🔍` | 预览窗口 | 开启后弹出 OpenCV 窗口，显示蓝框/黄框/绿框的匹配位置 |
| `○` / `⬤` | 调试截图 | 开启后自动保存匹配过程的截图到 debug_output 目录 |
| `□` / `▣` | 画面覆盖框 | 开启后把检测框直接画在游戏画面上 |
| `↻` | 重新框选 | 重新框选游戏区域（等同 Ctrl+Shift+R） |
| `⚙` | 设置面板 | 打开参数设置窗口 |
| `清空` | 清空记录 | 清空识别历史和统计数据 |
| `✕` | 隐藏面板 | 隐藏识别面板（可通过 Ctrl+Shift+H 重新打开） |
| `⏻` | 退出程序 | 完全退出程序（等同 Ctrl+Shift+Q） |

> **提示：** 按钮为绿色/青色时表示功能已开启，灰色表示关闭。鼠标在面板上的操作需要**按住 Alt 键**才能触发，防止干扰游戏。

## 快捷键

| 快捷键 | 功能 |
|--------|------|
| `Ctrl+Shift+R` | 重新框选 ROI |
| `Ctrl+Shift+H` | 显示/隐藏识别面板 |
| `Ctrl+Shift+Q` | 退出程序 |

## 识别历史面板

屏幕右侧的半透明面板，显示历史识别结果。

- 面板分为三个区域：**顶部功能栏** → **中部识别历史** → **底部计数区**
- 每条识别结果以 `[血脉] + [属性]` 的 chip 样式显示，例如 `[奇异] + [速度]`
- 底部计数区分左右两栏：血脉统计 和 属性统计
- 拖动标题栏可以移动面板位置
- 右下角拖拽可以调整面板大小
- 每个图案有独立颜色，可在设置的"提示文字"标签页中自定义

## 首次设置（重要）

### 1. 分辨率预设
打开 `⚙` 设置面板 → **Runtime** 标签页 → **游戏分辨率预设**，选择你游戏实际运行的分辨率：

| 预设 | 我的游戏窗口是 |
|------|--------------|
| 720p | 1280×720 左右 |
| 1080p | 1920×1080 左右 |
| 2K | 2560×1440 左右 |
| 4K | 3840×2160 左右 |

**注意**：选游戏窗口的实际分辨率，不是显示器的物理分辨率。

### 2. 检查黄框
蓝框命中后，预览窗口会出现两个黄框。确认黄框是否对准了你要识别的图案。如果偏了，在设置面板的 **黄框1 / 黄框2** 标签页调整位置。

## 设置面板

`⚙` 打开，各标签页功能：

| 标签页 | 改什么 |
|--------|--------|
| Anchor | 蓝框（盒子整体）的匹配参数、粗筛阈值和模板列表 |
| 黄框1 | 第一个图案区域的位置和大小 |
| 黄框2 | 第二个图案区域的位置和大小 |
| 样本1 | 第一个图案的模板、匹配阈值和缩放范围 |
| 样本2 | 第二个图案的模板、匹配阈值和缩放范围 |
| Runtime | 分辨率预设、帧率、归一化、跳帧间隔、宽度缩放匹配 |
| Debug | 预览窗口、调试截图、框线绘制开关 |
| 悬浮窗 | （已废弃，功能已合并到识别面板） |
| 提示文字 | 面板标题、字号、颜色、计数区设置 |

**改完记得点"保存并应用"。**

## 手动添加样本

如果需要识别新的图案，可以手动截取模板图片添加到项目中。

### 方法一：通过设置面板添加（推荐）

1. 打开 `⚙` 设置面板 → **样本1**（或 **样本2**）
2. 在下拉菜单中点击 `+ 新增图案`，输入图案名称
3. 选中新图案，点击 `+ 添加模板`，选择截好的模板图片
4. 根据需要调整该图案的匹配阈值和缩放范围
5. 点击 **保存并应用**

### 方法二：手动放置文件

1. 将模板图片放入对应文件夹：
   - 样本1 的模板 → `templates/patterns/`
   - 样本2 的模板 → `templates/patterns_2/`
2. 打开 `config.json`，在对应图案的 `templates` 列表中添加文件名
3. 重启程序

## 常用设置与识别效果的关系

### 匹配阈值

| 设置位置 | 参数名 | 含义 | 建议值 |
|---------|--------|------|--------|
| Anchor 标签页 | 匹配阈值 | 蓝框匹配的最低分数 | 0.65 ~ 0.75 |
| 样本1/样本2 | 匹配阈值 | 图案匹配的最低分数 | 0.60 ~ 0.70 |

- **调低** → 更容易匹配到，但可能误识别
- **调高** → 匹配更严格，但可能漏掉正确结果

### 缩放范围

| 设置位置 | 参数名 | 含义 | 建议值 |
|---------|--------|------|--------|
| Anchor 标签页 | 最小/最大缩放 | 蓝框的缩放搜索范围 | 取决于分辨率预设 |
| 样本1/样本2 | 最小/最大缩放 | 图案的缩放搜索范围 | 取决于分辨率预设 |
| Runtime 标签页 | 宽度缩放匹配 | 对抗透视畸变的宽度压缩 | 默认开启，3 个最宽模板 |

- **缩放步数**越多 → 匹配越精细，但越慢

### 粗筛阈值（Anchor 标签页）

- 用单档 scale 快速过滤空帧，避免无盒子的帧做完整搜索
- 设为 **0** → 关闭粗筛，每帧都做完整匹配（更准但更慢）
- 建议 **0.55 ~ 0.65**，不要超过匹配阈值

### 分辨率预设（Runtime 标签页）

改变分辨率预设会自动调整 Anchor 和所有图案的缩放范围：

| 分辨率 | Anchor 缩放 | 图案缩放 |
|--------|------------|---------|
| 720p | 0.40 ~ 0.65 | 0.35 ~ 0.75 |
| 1080p | 0.60 ~ 0.90 | 0.50 ~ 1.05 |
| 2K | 0.75 ~ 1.25 | 0.65 ~ 1.35 |
| 4K | 1.20 ~ 1.80 | 1.00 ~ 1.70 |

### 跳帧与性能

| 设置 | 含义 | 建议 |
|------|------|------|
| Anchor 跳帧间隔 | 每隔 N 帧做一次蓝框搜索 | 1 ~ 2，卡顿时可调大 |
| 检测帧率 | 每秒截图次数 | 20 ~ 30，不需要超过游戏帧率 |

### 黄框位置

- **黄框1**（样本1 的截取区域）和 **黄框2**（样本2 的截取区域）的位置和大小决定了匹配时看到的画面范围
- 框太大 → 包含干扰背景，降低匹配精度
- 框太小 → 可能截不到完整图案
- 在预览窗口（🔍）中观察黄框是否对准目标

### 字号与颜色

在 **提示文字** 标签页可以调整：
- 面板标题、状态文字、识别结果 chip、计数区的字号
- 血脉颜色、属性颜色、默认颜色
- 调整后保存并应用即时生效，无需重启

---

## 注意事项

- **单人 / 双人抓宠**：单人战斗和双人组队战斗的相机视角不同，盒子的屏幕位置会有偏移。建议分别框选一次 ROI 并保存各自的 config，或者切换后按 `Ctrl+Shift+R` 重新框选。
- **游戏分辨率变更**：调整游戏分辨率（窗口大小）后，画面比例发生变化，**必须重新框选 ROI**，否则蓝框可能识别不到。框选后去 Runtime 确认分辨率预设匹配当前游戏窗口。
- **设置面板改完后务必点"保存并应用"**，否则重启后配置丢失。

## 常见问题

**Q: 蓝框一直识别不到？**
- 检查 Runtime → 分辨率预设是否选对
- 调整 Anchor 标签页的"匹配阈值"（降低到 0.55-0.60）
- 调整"粗筛阈值"（降低或设为 0 关闭）

**Q: 蓝框识别到了但绿框匹配不准？**
- 降低样本1/样本2 的"匹配阈值"
- 调整黄框1/黄框2 的位置使其对准目标图案
- 增加模板样本的数量和多样性

**Q: 识别面板不见了？**
- 按 `Ctrl+Shift+H` 切换显示/隐藏

**Q: 游戏全屏时面板看不见？**
- 游戏设为无边框窗口模式即可

**Q: 鼠标无法点击面板按钮？**
- 按住 **Alt 键** 再操作面板，松开 Alt 后面板恢复穿透

**Q: 预览感觉卡顿？**
- Runtime → Anchor跳帧间隔 调大（如 2 或 3）
- 关掉预览窗口（面板顶部 ◌ 按钮）
- 关掉画面覆盖框（面板顶部 □ 按钮）
- 关掉调试截图（面板顶部 ○ 按钮）

**Q: 想要识别更多图案？**
- 样本1 / 样本2 标签页 → `+ 新增图案` → 添加模板图片 → 保存并应用
- 黄框2 默认关闭，需要先在黄框2 标签页勾选"启用区域二"

**Q: 程序打不开？**
- 不要放在中文路径下（虽然已尽量兼容，但仍可能有坑）
- 管理员权限运行试试
